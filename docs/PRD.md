# Hermes 桌宠 PRD

## 一句话说清

把本地的 Hermes agent CLI，包装成一个戴翅膀头盔的小机器人桌宠 —— 给它一张脸、几个入口、和一点"养着"的感觉。

## 愿景

让"用 agent"这件事从"打开聊天框打字"变成"桌面上有个伙计常驻"。它该在的时候在（工作模式藏角落不打扰），该陪着的时候陪着（普通模式有交互、有反馈）。功能上不比直接调 CLI 多什么，但**形态和氛围是核心价值**。

## 目标用户

第一阶段就一个：**Wayne 本人**，已有本地 Hermes agent 的开发者用户。MVP 不考虑分发。

## 核心概念

### 两种形态

桌宠有两种相互独立的状态。手动切换。状态指示靠头盔上的两只翅膀分开发光。

**普通模式（陪伴/休息）**
- 全身可见，待在桌面上
- 没事就原地转圈、扭头（每分钟一次 idle 动画）
- 鼠标长时间不靠近 → zzz 睡着
- 可以摸头（点击头部触发反馈动画 + 默默累计亲密度）
- 头盔左翅膀亮金色

**工作模式（高效/专注）**
- 钻进屏幕右上角，只露半个头 + 单侧翅膀
- hover 才浮出三个气泡：research / 对话 / cowork
- 头盔右翅膀亮蓝色
- 任务跑的时候涨红脸；做完了挥一下手

### 三个入口（仅工作模式可见）

| 气泡 | 干什么 |
|---|---|
| **research** | 调研类任务，system prompt 内置一个万能分析骨架（拆问题、列假设、找证据、给结论） |
| **对话** | 常规对话，可临时展开/收起 system prompt 编辑（改了只对当次有效） |
| **cowork** | 派单：交付一个完整任务 |

三个气泡的输出都不直接全部显示，**点开它**就能查看结果。

### 拖放

两种模式都能往桌宠身上拖东西（文件 / 文本片段 / URL），它会朝鼠标这边凑过来张嘴接住。工作模式下，拖动时 hover 哪个气泡，东西就进哪个气泡。

## 用户故事

### 故事 1 — 工作日常

> 我打开 IDE 写代码，把桌宠切到工作模式。它钻到右上角，右翅膀亮起蓝色。我在选中一段报错日志，拖到桌宠的 research 气泡上松开 —— 它叼着接住，涨红脸开始干活。两分钟后它挥了挥手提醒我，我点开 research 气泡就看到分析结果。

### 故事 2 — 摸鱼时刻

> 长会议结束我去倒咖啡，回来发现桌宠在原地 zzz 睡着了。我点了一下它的头，它睁眼眨了眨，扇了扇翅膀，像是在跟我打招呼。亲密度被默默 +1，我并不知道，但下次它在 idle 时会更倾向于看向我。

### 故事 3 — 临时调一下人格

> 我在跟它对话讨论一个文案方向，但默认 system prompt 太严肃。我点开对话气泡里的 system prompt，临时改成"用一个写广告文案 20 年的老炮的语气"，发了几条对话。改完不保存，下次再开对话气泡时还是默认那个。

## 范围

### MVP（V1）必须有

- 两种模式 + 手动切换 + 翅膀颜色指示
- 工作模式三个气泡（research / 对话 / cowork）能正常发起任务并取回结果
- 后端跑通：spawn 本地 Hermes CLI 子进程，流式拿输出
- 普通模式基础动画（idle、睡眠、摸头反馈）
- 拖放接住文件 / 文本 / URL
- 任务运行 → 涨红脸；完成 → 挥手；展开看结果
- 设置面板：退出、CLI 配置、prompt 默认值

### MVP 不做（推到 V2+）

- 多屏幕支持
- 桌宠在屏幕里走动
- 亲密度的 UI 可视化（V1 只默默记，不展示）
- 分发 / 多用户 / 账号体系
- 自定义皮肤、主题
- 跨会话上下文持久化（待定，看实现成本）

## 成功标准（自用）

- 我每天工作时它都开着且没把我关掉
- 我至少每天用一次三个气泡之一
- 切到普通模式后我会忍不住摸它一下
