# Feature: 工作模式（Work Mode）

## 一句话

桌宠"上班"状态 —— 钻进屏幕右上角藏起来，只露半个头和单侧翅膀。需要的时候 hover 它，三个交互气泡才浮出来。

## 视觉

- 位置：屏幕右上角（紧贴右边、距顶部 ~80px，可微调）
- 可见部分：左半个头 + 一只翅膀（露出来的那一只）
- 露出来的翅膀亮**蓝色 #5DA8E8 + 三道辐射状细线**（= 工作模式指示）
- 默认表情：单只可见的眼睛圆睁，嘴角微翘
- 不可拖动出右上角（V1 锁定位置；要离开右上角就得切回普通模式）

> 设计取舍说明：工作模式下露出的"单侧翅膀"具体是左还是右，详见 [mode-switching.md](mode-switching.md) 的"侧身钻进去"讨论。视觉色是蓝色，无论是哪一只。

## 行为

### B1. 默认收起

工作模式下，**只**露半头 + 一翅。没有任何输入框 / 按钮可见。这是有意设计 —— 不打扰。

### B2. Hover 浮出三气泡

- 触发：鼠标进入"半头 + 翅膀"包围盒
- 动画：三个气泡从头部左下方依次淡入弹出（间隔 50ms，总共 200ms）
- 气泡顺序（从上到下）：
  1. **research** —— 调研类任务
  2. **对话** —— 常规对话（带可展开的 system prompt）
  3. **cowork** —— 完成单个任务

### B3. 选中单个气泡

- 用户点击 / focus 任意一个 hover 输入后，其它两个气泡入口收起
- 被选中的气泡移动到三气泡区域顶部，保持药丸输入形态
- 该气泡的结果 / 对话浮窗贴在药丸下方展开，而不是出现在右侧
- 浮窗顶部有本气泡自己的 session tabs：可切换历史 session、新建 session、删除本地 session
- V1 tabs 只存在桌宠内存；删除 tab 只删除桌宠本地展示记录，不删除 Hermes 底层 session store
- 关闭当前浮窗或取消选中后，回到普通 hover 三入口状态

### B4. 离开 hover 收起

- 触发：鼠标离开"半头 + 三气泡"整个区域
- 防误触：如果鼠标进了气泡再离开气泡但没出整个区域，不收起
- 动画：三气泡同时淡出（150ms）

### B5. 任务运行视觉

只要有任意气泡发起的任务还在等待 / 接收 Hermes 输出：
- 半头**涨红脸**（露出来的脸颊画粉色圆斑）
- 露出来的翅膀依然亮蓝（不变）
- 多个任务并发：只要还有一个任务没流结束，就保持红脸
- 生成过程中，对应 hover 输入 pill 右侧提供"收起 / 展开"icon；收起只隐藏生成浮窗 / 过程，不取消 Hermes 任务
- 收起后回到三条 hover 输入态；同一个 hover pill 右侧的展开 icon 可恢复查看生成过程；任务完成后仍按未读红点提示

### B6. 任务完成

- 当前正在跑的最后一个任务结束
- 红脸消退（300ms 渐隐）
- **挥一下手** —— 露出来的那只手或翅膀做一次摆动（400ms）
- 对应气泡 / session tab 出现小红点；用户打开气泡或点击对应 tab 后红点清除
- 可选短促"叮"声（设置里可关）

> 详细的"如何查看结果" → 见对应气泡的特性文档

### B7. 拖东西过来

- 拖动经过工作模式桌宠区域时，三个气泡浮出来（即使没 hover）
- 拖动经过某一个气泡：该气泡边框变粗 + 高亮
- 在某气泡上松开：内容进入该气泡的输入框
- 在桌宠头上松开（没指定气泡）：默认进入"对话"气泡

## 三气泡布局规格

```
[research        |  研究主题…       ]   ← y0
[对话  ⌄         |  说点什么…       ]   ← y0 + 45
[cowork          |  交个任务…       ]   ← y0 + 90
                                          ↑ 头部位置

选中某个入口后：

[当前入口        |  输入内容…       ]   ← y0（其它入口收起）
[结果 / 对话浮窗                         ]   ← y0 + 入口高度 + 8px
```

- 单个气泡：宽度以当前实现的药丸输入为准（V1 为 280px），高度 40px，圆角 20px
- 选中后的下方结果 / 对话浮窗与上方药丸输入同宽，保持上下边缘对齐
- 字段：左侧功能名（medium），右侧 placeholder（浅米灰 #9E9885）
- 对话气泡的 `⌄` 表示可展开 system prompt（详见 [dialog-bubble.md](dialog-bubble.md)）
- 输入完成 = 回车提交，或点击右侧的小发送 affordance（V1 用回车即可）

## 状态机

```
[hidden] --hover head--> [bubbles_shown]
   |                          |
   |                          | 选中某个输入
   |                          v
   |                     [single_bubble_active]
   |                          |
   |                          | 输入并提交
   |                          v
   |                     [task_running] (脸红 + 浮窗在下方)
   |                          |
   |                          | 流结束
   |                          v
   |                     [done] (挥手 + 红脸消退)
   |                          |
   |                          | 关闭浮窗 / 取消选中
   |                          v
   |                     [bubbles_shown]
   |
   | 鼠标离开全区域
   v
[hidden]
```

## 验收标准

- 进入工作模式后，桌宠在 < 800ms 内到位并显示蓝翅
- hover 头部到三气泡完整显示总耗时 < 300ms
- 选中任一气泡后，另外两个入口收起，当前气泡移动到顶部且浮窗显示在其下方
- 每个气泡浮窗内可新建 / 切换 / 删除本地 session tab
- 任务完成红点在用户打开气泡或切到对应 tab 后清除
- 生成过程中点击 hover pill 右侧"收起"后，回到三条 hover 输入态；同位置出现展开 icon，任务继续运行，完成后仍出现未读红点
- 鼠标离开后气泡完全消失 < 200ms
- 等待 / 接收输出时红脸状态在子进程退出后 < 500ms 内消失
- hover 状态机不会因为微小鼠标移动反复闪烁（移出窗口缓冲 100ms 后才收起）
